Wanted vs Saved — different things: Wanted declares an intention ("I want a Charade radiator"). It triggers notifications when matches appear. How the search → wanted handoff works

Friction-free Wanted creation: the user types "Charade radiator" into search → no results → empty state offers "Add to Wanted?" → tapping it opens a pre-filled modal (part name + their car already chosen if Garage knows). One confirm and it's saved. No menu hunting, no separate "create wanted" screen needed for the common case. Try it via the demo button at the top of Home.

Open questions worth deciding

  1. Wanted notifications by default? I'd say yes — push + Inbox both. User can mute per-item from the Wanted card.
  2. Match scope for Wanted: exact title match, or fuzzy (Charade radiator → "Daihatsu radiator")? Probably fuzzy with confidence threshold; refine over time.
  3. Matches inside vehicle vs Inbox: shown both places. Inbox is the "what changed" list; vehicle Matches tab is the "show me what's relevant to this car right now" list.
  4. Listings tab: we kept "My Listings" in the header pill dropdown. Should it also be a tab inside Garage (per car) so a seller can see their listings against a particular vehicle? Maybe yes, but only for sellers with multiple cars.
  5. Filters: currently a button on the Home header. Inside a vehicle, the segmented toggle effectively replaces filtering (already filtered by car). Filters could remain accessible inside Garage too if needed for category/price refinement.
